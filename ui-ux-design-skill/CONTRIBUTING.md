# 🤝 Contribuindo

Obrigado por querer contribuir! Este projeto melhora com a experiência de designers e devs do mundo real.

## Como Contribuir

### Reportar um Bug
Abra uma [Issue](../../issues/new?template=bug_report.md) descrevendo:
- O prompt que usou
- O resultado recebido vs. esperado
- Contexto (framework, tipo de projeto)

### Sugerir uma Melhoria
Abra uma [Issue](../../issues/new?template=feature_request.md) descrevendo:
- Qual recurso está faltando
- Casos de uso concretos
- Exemplos de referência (links de sites, codepens etc.)

### Pull Request

1. Fork o repositório
2. Crie uma branch: `git checkout -b feat/nome-da-feature`
3. Faça suas alterações
4. Commit: `git commit -m "feat: descrição clara da mudança"`
5. Push: `git push origin feat/nome-da-feature`
6. Abra um Pull Request

### O que aceito como PR

✅ **Aceito:**
- Novos padrões de componentes com código de qualidade
- Novas paletas de cores com sistema completo
- Correções de erros no código de exemplo
- Melhorias na documentação
- Traduções

❌ **Não aceito:**
- Código genérico sem identidade
- Exemplos que violam as boas práticas da skill
- Dependências desnecessárias

## Padrão de Código

Todo código de exemplo deve seguir:
- Comentários explicativos quando não óbvio
- Funcionar em browsers modernos (últimos 2 anos)
- Incluir versão mobile quando aplicável
- Respeitar `prefers-reduced-motion`
- Sem warnings no console

## Convenção de Commits

```
feat:     nova funcionalidade
fix:      correção de bug
docs:     mudança na documentação
refactor: refatoração sem mudança de comportamento
perf:     melhoria de performance
```
